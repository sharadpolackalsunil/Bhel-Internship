⚡ Project Overview

Why This Project Stands Out

University portals are notoriously difficult to scrape. This project solves complex engineering challenges with cutting-edge techniques.

🕸️

Web Automation

Selenium WebDriver handles ASP.NET ViewState, dynamic PostBacks, JavaScript radio buttons, and session-bound dropdowns that break traditional HTTP scrapers.

🧠

Deep Learning OCR

Microsoft's TrOCR (Vision Transformer + RoBERTa decoder) achieves ~99% accuracy on distorted CAPTCHAs — no manual labeling, no custom CNN training required.

📊

Data Engineering

Pandas-based processor automatically flattens nested HTML tables, handles missing data, generates grade columns dynamically, and exports formatted Excel workbooks.

🛑 Engineering Challenges

Problems We Solved

Building this system required overcoming 4 critical engineering challenges.

Problem

Custom-trained CNN had only 10-20% accuracy on the portal's highly distorted and noisy CAPTCHAs, causing constant scraper failures.

Solution

Replaced CNN with Microsoft's TrOCR (Transformer-based OCR) — a Vision-Encoder-Decoder architecture pre-trained on millions of printed text images. Accuracy jumped to ~99% instantly.

Problem

Even with 100% correct TrOCR predictions, the portal kept rejecting CAPTCHAs. The server said "incorrect" despite our text being right.

Solution

ASP.NET uses hidden __VIEWSTATE variables. Our scraper was bypassing the program selection page, creating an invalid session. Fixed by navigating through ProgramSelect.aspx → clicking B.Tech radio → selecting semester → injecting JavaScript to force-reload the CAPTCHA with a unique timestamp to bust the browser cache.

Problem

The exported CSV was a mess — generic text in the Program column, missed SGPA entirely, and all subjects lumped into generic Subj_1 columns.

Solution

Analyzed the exact DOM structure and rewrote parse_result_page() with BeautifulSoup. Targeted specific ASP.NET lbl IDs for header info, accurately extracted the Grades table with Course Codes/Credits/Grades, and targeted the Footer table for SGPA/CGPA/Pass-Fail status.

Problem

The git push command failed completely when trying to push the finished project to GitHub.

Solution

The TrOCR model file (model.safetensors) was 1.33 GB — far exceeding GitHub's 100 MB limit. Performed git reset to undo commits, added the model directory to .gitignore, and recreated the commit cleanly.
